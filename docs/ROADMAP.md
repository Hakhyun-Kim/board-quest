# 앞으로 할 만한 것 (Roadmap)

우선순위대로. 각 항목은 `docs/DESIGN.md`의 **확장 판단 기준 3가지**(절차 생성 우선 · 재미 · 검증 가능)를 통과한 것들이다.
짧게 끝나는 것과 오래 걸리는 것을 섞어 두었으니, 한 번에 하나씩 골라 커밋 단위로 끝내면 된다.

## 지금 바로 (코어를 굵게)

### ~~1. 스킬 시스템~~ ✅ 완료 (v0.3)
직업당 특기 1종, 쿨다운제로 구현. 회전베기 · 저격 · 기도 · 관통. `src/lib/skills.ts` + `battle.ts`의 `doSkill`.
남은 갈래는 아래 **2번(적도 특기를 쓰게)**, 그리고 직업당 2번째 특기.

### 1. 밸런스 시뮬레이터 ★지금 최우선
헤드리스로 전투 N판 자동 진행 → 승률 · 평균 라운드 수 · 사망 분포 리포트.
- **왜 지금인가**: 특기가 들어오면서 **재야 할 수치가 생겼다.** 쿨다운 3/4가 적정한지, 저격 1.5배가 센지, 회전베기가 검사를 너무 강하게 만들진 않는지 — 지금은 전부 추측이다.
- **어떻게**: `src/dev/simBot.ts` — 브라우저 콘솔에서 `__bqsim.start({ tier: 3, runs: 200 })`. 렌더 없이 `createBattle` → `foeAct` + 간이 아군 AI 루프.
- **참고**: 특기 검증 때 쓴 방식이 그대로 쓸 만하다 — `npx esbuild <테스트>.ts --bundle --platform=node --format=esm` 로 묶어 `node`로 돌리면 브라우저 없이 순수 로직을 굴릴 수 있다.

### 2. 적도 특기를 쓰게
지금 특기는 **아군 전용**이라, 적은 여전히 기본 공격만 한다.
- **왜**: 플레이어만 새 도구를 쥐면 난이도가 한쪽으로 기운다. 보스가 특기를 쓰면 마지막 전투가 실제로 "벽"이 된다.
- **어떻게**: `foeAct`의 (칸 × 표적) 평가에 스킬 후보를 추가 — 이미 `skillVictims`로 맞을 대상을 뽑을 수 있으니 점수만 매기면 된다. 오크 대장에게 전용 특기(포효: 주변 아군 공격 버프)를 주면 보스전이 살아난다.
- **세트로**: 1번 시뮬레이터가 있으면 "적에게 스킬을 줬더니 승률이 얼마나 떨어졌나"를 즉시 볼 수 있다.

### 2.5. 방향키 칸 커서 (조작 되돌리기)
이동·공격을 메뉴에서 빼면서 **키보드만으로는 이동·공격을 할 수 없게 됐다.** 방향키로 칸 커서를 움직이고 Enter로 확정하는 경로가 필요하다.
- **왜**: `CLAUDE.md`의 "모든 선택 화면은 방향키로 다룰 수 있다" 규약을 전투 판만 못 지키고 있다.
- **어떻게**: 커서 칸 상태 하나 + 방향키 핸들러. 커서 칸을 노란 테두리로 그리고, Enter는 `onTile(cursor)`를 그대로 부르면 된다 — 마우스와 같은 경로라 로직 추가가 거의 없다.

### 3. 전투 연출 보강
이동 경로 따라 미끄러지기 · 피격 번쩍 · 데미지 숫자 튀어오르기 · 처치 연출 · 카메라 살짝 흔들기.
- **왜**: 지금은 위치 보간만 있어서 "판정이 일어났다"는 감각이 약하다. 로직을 안 건드리고 체감을 크게 올리는 가성비 항목.
- **주의**: 연출은 **로직과 분리**할 것 — 연출을 기다리느라 상태 전이가 막히면 시뮬레이터가 못 돌린다.

## 그다음 (전투를 깊게)

### 4. 적 AI 고도화
지형 선점(숲·언덕) · 집중 공격 · 사제 우선 노리기 · 열세면 후퇴.
- 1번 시뮬레이터가 있으면 "AI를 바꿨더니 승률이 어떻게 변했나"를 즉시 볼 수 있어 개발이 훨씬 빨라진다. **1번 이후에 하는 게 이득.**

### 4.5. 직업당 두 번째 특기
지금은 직업마다 1종. 2종이 되면 "이번 턴에 어느 쪽을 쓸까"가 생겨 빌드 다양성이 한 겹 더 붙는다.
- 데이터만 추가하면 되는 구조다 (`SKILLS`에 항목 추가 + `ClassDef.skill`을 배열로). 다만 **1번으로 현재 4종의 균형을 먼저 잰 뒤**에 늘리는 게 순서다.

### 5. 진형/배치 단계
전투 시작 전 아군 시작 칸을 직접 고르는 단계.
- 구현량이 작은데 의사결정이 하나 늘어난다. 가성비 좋음.

### 6. 지형 상호작용
높이차에 따른 사거리·피해 보정 · 물가로 밀어내기 · 파괴 가능한 바위.

## 원정 층 (지도를 심심하지 않게)

### 7. 원정 사건 노드
갈림길에 이벤트 노드 — 용병 제안 · 함정 · 수수께끼 · 상인. 백층 던전의 방 이벤트와 같은 계열.

### 8. 저장 / 이어하기
원정 중간 상태를 `bq-run` 키로 localStorage에. 모바일에서 탭이 날아가도 이어갈 수 있게.

### 9. 동료 개성 · 상실
절차 생성 이름 + 소소한 특성. 죽으면 영영 잃는 규칙을 넣으면 원정에 감정적 무게가 생긴다.

## 마감 · 곁가지

### 10. 음악
절차 생성 BGM(지도 / 전투 / 승리). 백층 던전 `music.ts` 패턴 이식 — 외부 에셋 0개 원칙 유지.

### 11. 모바일 최적화
터치 타겟 크기 · 세로 화면 카메라 맞춤 · DPR 제한(고해상도 기기 발열 방지).

### 12. 바둑 · 오목식 미니게임
마을이나 특수 노드에서 즐기는 판 위 미니게임. 원작 영감 회수용.

### 13. 번들 크기 줄이기
현재 JS 번들 약 1.0MB(gzip 292KB) — 대부분 three.js다. 첫 로딩이 신경 쓰이면 three 모듈을 골라 import 하거나 타이틀 화면 이후로 동적 import.

## 어디부터 손댈지 고민된다면

**1번(시뮬레이터) → 2번(적 특기)** 순서를 권한다. 특기가 들어오면서 재야 할 수치가 생겼는데 지금은 전부 추측이고,
측정 도구가 먼저 있으면 그다음 밸런스 작업(적 특기·AI·2번째 특기)이 전부 빨라진다.
눈에 보이는 변화가 급하면 3번(연출)이 로직을 안 건드려서 안전하다.
